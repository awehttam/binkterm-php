BinktermPHP 1.8.8 Released


BinktermPHP 1.8.8 is out! This release focuses on message rendering quality,
QWK offline mail improvements, and a number of usability refinements across
the web interface.

--- MESSAGE READER IMPROVEMENTS ---

The biggest area of work this release is how messages look on screen.

Space-aligned ASCII art and tables have been improved. Plain text messages
previously collapsed consecutive spaces to one, breaking any content that
relied on spacing for alignment. Fixed with white-space: pre-wrap on message
body lines.

Line spacing has been tightened across all themes to match the closer cadence
of BBS messages rather than web-article prose.

ANSI art auto-detection is now cleaner: it triggers only on actual ANSI escape
sequences or pipe colour codes. The old leading-space heuristics are gone
since they are no longer needed.

PETSCII has been removed as a message art format option. It is a binary format
that cannot survive FTN text transport. The renderer stays in place for file
area previews of .prg and .seq files.

--- SENDER NAME POPOVER ---

Clicking a sender name anywhere in echomail or netmail now shows a popover
with the BBS system name, location, and FTN address pulled from the nodelist.
Point addresses are resolved to the boss node. Buttons in the popover let you
jump straight to sending a netmail reply or viewing the full nodelist entry.

--- COMPOSE IMPROVEMENTS ---

The To Address field in netmail compose now has nodelist autocomplete. Start
typing a sysop name, system name, location, or partial FTN address and a
dropdown appears with matching nodes.

The To Name field now supports address book autocomplete. Matching contacts
show their node address, network name, and BBS name from the nodelist.
Selecting an entry fills both the To Name and To Address fields.

The Encoding and Markup Format selectors have been moved into a collapsible
Advanced Options panel that sits below the message textarea, toggled by a
button next to Insert File. The panel stays open or closed between sessions.

A Hard Wrap option inside that panel lets you choose automatic line breaking
at 79 characters (standard 80-column format), 39 characters (for readability
on 40-column systems like the C64), or off if you prefer to manage line
length yourself. Wrapping happens at word boundaries where possible. The
setting is remembered per user.

The compose form now warns you if your message is approaching or has exceeded
the 16 KB FidoNet message limit. A yellow warning appears as you type once
you pass 80% of the limit, and a red error if you go over. The same checks
run at send time.

--- CHARACTER SET HANDLING ---

Replies now honour the charset of the message you are replying to. Replying
to a CP437 message sends CP437. If the reply body contains characters that
cannot be represented in that charset it falls back to UTF-8 automatically.
A new Encoding selector in the compose form lets you override the charset
manually. CP1252 has been added as a supported outgoing option.

The Art Encoding field in message edit dialogs has been renamed to Character
Set and changed from a free-text input to a validated dropdown.

--- QWK OFFLINE MAIL ---

Several QWK interoperability fixes: conference numbers are now stable
BBS-wide IDs rather than subscription position, QWKE Subject headers are
now read from MultiMail-style REP packets fixing 25-character truncation,
and From/To extended headers no longer include the FTN address appended to
the name. New netmail via QWK now supports zone:net/node addressing in the
To field. The QWK BBS ID is now a configurable setting.

--- LOVLYNET ---

The LovlyNet admin Setup tab now has a checklist that runs five checks in
parallel: registration status, hub connectivity, uplink configuration,
default area subscriptions, and the nodelist file area rule. A Fix button
can automatically create the nodelist file area rule if it is missing.
New uplinks added by the setup script now default to CRAM-MD5.

--- WEB INTERFACE ---

- Today's Callers list added to the sysop dashboard
- Activity Stats login breakdown by source (web / telnet / SSH)
- Profile page now shows network information and file transfer stats
- Admin settings pages blur while loading and show a spinner
- Sysop/admin profile now has an Activity Log tab
- Broadcast Manager (formerly Ad Campaigns) now has a Weather Report preset
- Weather Report configuration page added under Admin -> Community
- Ad/bulletin content rendering now uses the same multimodal pipeline as
  the message viewer (RIPscrip, Sixel, ANSI, PCBoard, plain text)
MRC chat received several improvements: tab-completion for usernames (press
Tab to complete, again to cycle through matches) and for slash commands
including /msg with username completion. A sidebar toggle lets users switch
between simple polling and long polling with the preference saved locally.
Bug fixes: first message in a new DM conversation now appears immediately,
room user count was off by one, sent DM messages no longer disappear when
there is no prior conversation history, and the default room is shown when
the server returns an empty room list.
